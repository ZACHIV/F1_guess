# F1引擎声猜赛道 核心开源仓库汇总

以下是精选的**7个核心开源项目**，覆盖数据获取、音频处理、赛道可视化、声纹识别等全流程，可直接复用快速搭建你的网页/小程序：

## 一、F1数据获取类（核心基础）

### 1. Fast-F1（Python库）
- **链接**：https://github.com/theOehrly/Fast-F1 
- **核心功能**：获取F1官方遥测数据（GPS坐标、RPM、速度、时间戳）、比赛结果、赛程信息
- **适用场景**：为每个赛道生成精准的坐标序列，与音频时间戳对齐实现赛车位置实时同步
- **特点**：与Pandas无缝集成，支持2018年至今所有比赛数据，社区活跃（4.1k+ stars）

### 2. OpenF1 API
- **链接**：https://github.com/openf1/openf1 
- **核心功能**：RESTful API接口，提供实时遥测数据（位置、速度、油门、刹车、RPM、档位，3.7Hz采样率）
- **适用场景**：小程序/网页端直接调用，无需后端Python环境
- **特点**：开源免费，支持WebSocket实时推送，官方文档完善

## 二、音频处理与声纹识别类

### 3. Formula1 F1 Pure Engine Audio Dataset（Kaggle数据集）
- **链接**：https://www.kaggle.com/datasets/govindamadhava/formula1-f1-pure-engine-audio-dataset-2012-2024 
- **核心内容**：2012-2024年F1纯引擎车载音频（WAV格式），无解说干扰
- **适用场景**：直接作为项目音频数据源，无需自行提取
- **特点**：高质量音频，覆盖多赛季、多赛道，已分类整理

### 4. Engine-Sim（发动机声音模拟器）
- **链接**：https://github.com/ange-yaghi/engine-sim 
- **核心功能**：生成逼真的内燃机声音，支持自定义参数（气缸数、排量、转速）
- **适用场景**：补充官方音频数据，或生成特殊场景下的引擎声
- **特点**：实时仿真，支持导出WAV文件，可用于音频增强或测试

## 三、F1可视化与前端类

### 5. StarTrack F1（Web应用）
- **链接**：https://github.com/poleposition-labs/startrack-f1 
- **核心功能**：赛道设计、策略建模、遥测数据分析的Web应用
- **适用场景**：复用赛道可视化组件，实现赛车在2D/3D赛道上的实时移动
- **特点**：基于物理引擎的车辆模型，支持自定义赛道，前端代码可直接参考

### 6. F1-Dash（遥测数据可视化仪表盘）
- **链接**：https://github.com/rhtnr/f1-dash 
- **核心功能**：Python+D3.js构建的F1遥测数据可视化Web应用
- **适用场景**：学习如何将FastF1数据与前端可视化结合，实现赛车位置实时更新
- **特点**：支持圈速分析、轮胎策略可视化、遥测数据图表，有在线演示（https://f1-dash.live/）

## 四、声纹识别与音频匹配类

### 7. SonicSig（音频识别系统）
- **链接**：https://github.com/UjjWaL-0911/SonicSig 
- **核心功能**：结合传统音频指纹识别（Shazam风格）和深度学习（YAMNet嵌入）实现音频识别
- **适用场景**：实现引擎声音到赛道的自动匹配，提升项目AI能力
- **特点**：支持上传音频片段识别，生成音频指纹，可自建参考音频数据库

## 快速使用建议
1. **基础版**：先用Fast-F1获取赛道坐标 + Kaggle音频数据集，结合我们之前的前端代码快速搭建MVP
2. **进阶版**：集成SonicSig实现自动声纹识别，或用OpenF1 API实现实时数据更新
3. **音频提取**：用FFmpeg（https://ffmpeg.org/）从F1官方车载视频提取纯引擎声：
   ```bash
   ffmpeg -i input.mp4 -vn -acodec pcm_s16le -ar 44100 -ac 2 output.wav
   ```
